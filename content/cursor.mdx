# Cursor

这篇文章讲解如何使用cursor帮助我们进行代码编写，分享一些cursor的使用技巧以及注意事项，适合零基础的成员进行阅读

cursor作为目前模型支持最多，功能最丰富的AI IDE，受到了了许多开发者的青睐，其内置的强大AI模型，可以让编程小白也能快速地做出自己的项目，体验到AI编程的乐趣。

首先访问Cursor的官网：

[https://www.cursor.com/](https://www.cursor.com/)

![cursor1.png](/blogs/cursor/cover.png)

进入官网后，点击下载按钮开始下载，待下载完成后，进行安装即可开始使用

第一次使用时需要进行注册，免费用户有14天的pro体验，到期后可选择是否订阅

Pro用户需要美月支付20美元订阅费，将获得每月500次的快速提问，使用最新的模型等

（个人推荐购买）

进入应用后，将会看到以下界面

![](/blogs/cursor/01.png)

左边是文件管理栏，中间是编写代码的地方，最右边是AI聊天框

（该应用还未上线中文，英文不好的用户体验不会很好，不过会慢慢习惯）

可以先在右边的聊天框中让AI创建一个贪吃蛇小游戏

![](/blogs/cursor/02.png)

![](/blogs/cursor/03.png)

AI会自动帮忙创建文件并写入代码，用户只需执行相应的命令即可

将鼠标移动到AI提示的命令处，会出现‘Apply’的按钮，按下后将开始执行命令

还可以在这里选择AI的回答模式：Agent，Ask，Manual

![](/blogs/cursor/04.png)

Agent模式可以端到端地完成一个任务，可以将代码直接应用到编码区中，在项目初期，可以使用该功能进行项目的搭建

Ask模式可以询问当前文件下代码相关的问题，以便更好地查找错误

当然可以使用‘@+文件名’引用目标文件，让AI更好地知道你想要分析的文件

在Agent模式下，如果项目文件内容较多，在询问时没有@相关的目标文件，Cursor会将项目的整体框架进行阅读，无法更精确地给出代码的构建，因此在项目内容较多的时候，需要使用‘@+文件名’来引用目标文件，以便更好地构建项目和修改问题

在这里可以进行模型的选择

![](/blogs/cursor/05.png)

Cursor支持很多优秀的模型，如：GPT4o，Claude3.7，DeepSeek，Gemini等

可以多多进行其他模型的尝试，选择一个更适合自己的模型

结合我个人的使用体验，给出以下不同模型编写代码以及回答时的特色：

Gemini2.5-Pro：分析的很详细，并且会给出很全面的解释，编写的代码也会有很全面的注释

claude3.7-sonnet：响应速度很快，思考时间少，在进行前端设计时会有很棒的页面UI

o3-mini：对代码错误的修改很全面

DeepSeek：前端设计很优秀，编写独立程序时有较好的表现，代码修改较少

如果说只进行简单的模型选择，还不足以编写出让我们满意的代码，有时候模型输出的格式，创建的文件，并不是我们想要的。

这时候就需要为模型编写额外的prompt，来让模型在编写代码时遵循我们指定的规则

点击右上角的设置

![](/blogs/cursor/06.png)

选择Rules

![](/blogs/cursor/07.png)

这里有两个地方可以编写Rules

一个是User Rules，这里编写的规则对所有的模型都生效，并且对每一个项目也会生效，类似于全局规则

另一个是Project Rules

点击Add new rule

![](/blogs/cursor/08.png)

输入项目规则的名称，回车后，就创建好了一个空的项目规则文件，文件以markdown格式进行存储，方便我们进行规则的编写

![](/blogs/cursor/09.png)

Rule Type处建议选择Always，这样当在一个项目中每开启一轮新的对话时都会将编写的规则应用到AI模型中，从而避免忘记引用规则的问题

例如：我想让ai编写代码时将每个函数的作用都注释在函数开头处，就可以编写一个简单的规则

![](/blogs/cursor/10.png)

保存完该规则后，下次代码生成时，就会有函数注释出现：

（编写规则后）

![](/blogs/cursor/11.png)

（编写规则前）

![](/blogs/cursor/12.png)

这样，在AI生成代码的时候，就可以按照我们的意愿或者编码习惯进行生成

如果自己并不知道如何更好地编写规则以达到更好地编码体验，可以访问以下项目地址：

[https://github.com/PatrickJS/awesome-cursorrules](https://github.com/PatrickJS/awesome-cursorrules)

该项目涵盖了各种不同语言编写时的代码规范，例如Next.JS， Vue，浏览器插件开发规范等等，可以帮助你有更加舒适的编码体验。
